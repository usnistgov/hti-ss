Description

At 2:35 PM, the nurse collects the patient’s height and weight, documenting it in the patient’s record. The provider examines the patient’s arm, and sends her for an x-ray of the affected area. revealing a fracture. The provider adds a diagnosis to the patient record indicating the fractured arm (ICD-10 CM S52.6- Fracture of lower end of ulna) as a working diagnosis. 

The facility`s electronic health record module for syndromic surveillance data assembles and transmits an Update ADT message about this patient encounter to the State Public Health Department (NIST PH SS FAC 2). The "full, legal patient name" and birth date is included due to state required reporting of all patients, which is not always the case across all jurisdictions.  


Comments

Dates and times are provided in this test scenario to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the test scenario with HIT, only date and time format will be validated within tester submitted test data.


Pre-condition

A04-Registration message is sent before the A08-Update message.


Post-Condition

An A08-Update message has been triggered.


Test Objectives

This test step examines a Health IT Module's ability to create an ADT^A08 Update message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019 conformance requirements.

  • Setting is Ambulatory.
  • Chief complaint is captured as narrative in PV2-3 Admit Reason
  • Patient Name is Identifiable in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and pseudonymized data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in an identifiable manner is required for this Test Step.
  • Diagnoses are submitted test using either ICD-10-CM or SNOMED-CT: Per ASTP/ONC, for certification testing  a Health IT Module can use either the ICD-10-CM or SNOMED CT® value sets in the submitted messages for all of the test steps in the Syndromic Surveillance Test Suite. Where the tool does not have test data that supports the Health IT Module’s value set (either ICD-10-CM or SNOMED CT®), the developer of the Health IT Module must provide the codes and testers must perform a visual inspection of the messages for these test steps to ensure that equivalent and valid ICD-10-CM or SNOMED CT® are used to populate the messages.  This policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). SNOMED-CT® codes are provided/required in the Test Data for coding the diagnoses in this Test Case.
  • Pediatric case where age is measured in months.
  • Add working Diagnosis
  • Multiple Races
  • Insurance known
