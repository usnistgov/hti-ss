Description

At 10 AM, the patient`s laboratory and radiology tests results are provided to the attending physician. The tests indicate that the infant has influenza and does not have pneumonia. The patient`s mother is provided with treatment information and prescriptions for her infant son. The working diagnoses (ICD-10 CM J11.1 Influenza due to unidentified influenza virus with other respiratory manifestations and H61.329 Acquired stenosis of external ear canal secondary to inflammation and infection, unspecified ear) are updated as final diagnoses. At 10:20 AM the patient is discharged home. 

NIST Hospital's Urgent Care is an outpatient facility operated by NIST Hospital that routinely sends electronic syndromic surveillance data to the the City Health Department (NIST PH SS FAC 1) in accordance with a city regulation. At 10:30 AM on February 20, 2025, NIST Hospital`s electronic health record module for syndromic surveillance data assembles and transmits a Discharge ADT message about this patient encounter on behalf of the Urgent Care facility as part of a routine that transmits new or updated content every 15 minutes. The patient name and birthdate are de-identified in the submission, so no date of birth is included, and the name is represented with name type 'S' to indicate that the name is empty for anonymity.


Comments

Dates and times are provided in this test scenario to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the test scenario with HIT, only date and time format will be validated within tester submitted test data. 

Automated triggering in this test case is an example for the test story, but there is no requirement that the System Under Test demonstrate support for such automated triggers.


Pre-condition

A08-Update message is sent before the A03-Disposition message.


Post-Condition

No Post-Condition


Test Objectives

This test step examines a Health IT Module's ability to create an ADT^A03 Disposition message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019 conformance requirements.

The following test objectives and variants are supported by this test case:

  • Setting is Urgent Care.
  • Chief complaint is submitted as free-text.
  • Patient name is de-identified in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and de-identified data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in a de-identified manner is required for this Test Step.
  • Diagnoses are submitted test using either ICD-10-CM or SNOMED-CT: Per ASTP/ONC, for certification testing  a Health IT Module can use either the ICD-10-CM or SNOMED CT® value sets in the submitted messages for all of the test steps in the Syndromic Surveillance Test Suite. Where the tool does not have test data that supports the Health IT Module’s value set (either ICD-10-CM or SNOMED CT®), the developer of the Health IT Module must provide the codes and testers must perform a visual inspection of the messages for these test steps to ensure that equivalent and valid ICD-10-CM or SNOMED CT® are used to populate the messages.  This policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). ICD-10-CM codes are provided/required in the Test Data for coding the diagnoses in this Test Case.
  • Pediatric case where age is measured in months.
  • Updates to Diagnoses
  • Multiple Races
  • Insurance is known
