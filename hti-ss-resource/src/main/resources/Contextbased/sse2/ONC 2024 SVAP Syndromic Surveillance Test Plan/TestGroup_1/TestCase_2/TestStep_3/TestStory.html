Description

The test results are available 4-days post-discharge confirming that the patient had a cryptosporidium infection. At 2PM, the physician reviews the test results and updates the patient's record with this new Cryptosporidiosis (disorder) diagnosis ((SNOMED-CT 240370009), and the working diagnosis 'Gastroenteritis (disorder)’ (SNOMED-CT 25374005) is updated as a final diagnosis.  

NIST Urgent Care is an independently operated outpatient facility that routinely sends electronic syndromic surveillance data to the State Department of Public Health (NIST PH SS FAC 2) in accordance with state regulation. At 6:00 PM, the facility`s electronic health record module for syndromic surveillance data assembles and transmits an Update ADT message with the working diagnoses and the height and weight information for this patient encounter to the State Public Health Department (NIST PH SS FAC 2). The "full, legal patient name" and birth date is included due to state required reporting of all patients, which is not always the case across all jurisdictions.  


Comments

Dates and times are provided in this test scenario to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the test scenario with HIT, only date and time format will be validated within tester submitted test data.


Pre-condition

A04-Registration and A03-Discharge messages are sent before the A08-Update message.


Post-Condition

Patient record is updated with procedures, diagnoses and patient height/weight. An ADT*A08 message has been generated with this updated content.


Test Objectives

This test step examines a Health IT Module's ability to create ADT^A08 Update message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019 conformance requirements.

The following test objectives and variants are supported by this test case:

  • Setting is Urgent Care.
  • Chief complaint is captured as narrative in PV2-3 Admit Reason.
  • Patient name is identifiable in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and pseudonymized data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in an identified manner is required for this Test Step.
  • Diagnoses are submitted test using either ICD-10-CM or SNOMED-CT: Per ASTP/ONC, for certification testing  a Health IT Module can use either the ICD-10-CM or SNOMED CT® value sets in the submitted messages for all of the test steps in the Syndromic Surveillance Test Suite. Where the tool does not have test data that supports the Health IT Module’s value set (either ICD-10-CM or SNOMED CT®), the developer of the Health IT Module must provide the codes and testers must perform a visual inspection of the messages for these test steps to ensure that equivalent and valid ICD-10-CM or SNOMED CT® are used to populate the messages.  This policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). SNOMED-CT® codes are provided/required in the Test Data for coding the diagnoses in this Test Case.
  • Adult case where age is measured in years.
  • Update Diagnosis Post-Discharge
  • Insurance is unknown
