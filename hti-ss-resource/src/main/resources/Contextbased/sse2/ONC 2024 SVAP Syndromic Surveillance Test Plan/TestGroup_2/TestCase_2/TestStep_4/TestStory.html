Description

While the patient is prepared for transport within the hospital for ECMO, the clinical staffers complete an admission record. The Reason for admission is "The Abdominal scan shows the newborn has Pericardial and diaphragmatic defect syndrome.". The Admit Diagnosis is recorded as, "Congenital diaphragmatic hernia (CDH)". 

NIST Hospital reports syndromic surveillance data to the City Health Department (NIST PH SS FAC 1). At 8:35 AM on February 1, 2025, the hospitals electronic health record module for syndromic surveillance data assembles and transmits an Admission message about this encounter. The "full, legal patient name" and birth date is included due to state required reporting of all patients, which is not always the case across all jurisdictions.  


Comments

Dates and times are provided in this test case to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the Test Case with an HIT Module, only date and time format will be validated within tester submitted test data.


Pre-condition

A03-Discharge message is sent before A01-Admission message.


Post-Condition

No Post-Condition


Test Objectives

This test step examines a Health IT Module's ability to create ADT^A01 Admission message within the HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019 conformance requirements for syndromic surveillance.

  • Setting is Emergency Department with Inpatient Admission.
  • Chief complaint is captured as narrative in PV2-3 Admit Reason.
  • Diagnoses are submitted test using either ICD-10-CM or SNOMED-CT: Per ASTP/ONC, for certification testing  a Health IT Module can use either the ICD-10-CM or SNOMED CT® value sets in the submitted messages for all of the test steps in the Syndromic Surveillance Test Suite. Where the tool does not have test data that supports the Health IT Module’s value set (either ICD-10-CM or SNOMED CT®), the developer of the Health IT Module must provide the codes and testers must perform a visual inspection of the messages for these test steps to ensure that equivalent and valid ICD-10-CM or SNOMED CT® are used to populate the messages.  This policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). SNOMED-CT® codes are provided/required in the Test Data for coding the diagnoses in this Test Case.
  • Patient Name is Identifiable in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and pseudonymized data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in an identified manner is required for this Test Step.
  • Multiple Races.
  • Pediatric case where age is measured in months.
  • Addition of Admitting Diagnoses.
  • Discharge Disposition Home.
  • Insurance is unknown
