Description

Two days post-visit, the patient`s laboratory tests results are provided to the pediatrician. The tests indicate that the patient has tested positive for measles and the physician updates the working diagnosis Measles (disorder) (SNOMED-CT 14189004) to a final diagnosis at 1pm. 

NIST Primary Care is an outpatient facility that routinely sends electronic syndromic surveillance data to the City Health Department (NIST PH SS FAC 1) in accordance with a city regulation. The state regulations permit sending identifiable patient information for the purposes of syndromic surveillance. in accordance with state regulation. The facility`s electronic health record module for syndromic surveillance data assembles and transmits an Update ADT message about this patient encounter at 6:00 PM. The patient name and birthdate are de-identified in the submission, so no date of birth is included, and the name is represented with name type 'S' to indicate that the name is empty for anonymity.


Comments

Dates and times are provided in this test scenario to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the test scenario with HIT, only date and time format will be validated within tester submitted test data. 

The dates within the message and across the messages in this test case will be subject to visual inspection to verify data consistencies and the ability to conform to the requirement to send this message within 12 hours of the event triggering this message.


Pre-condition

A03-Disposition message is sent before the A08-Update message.


Post-Condition

An A08-Update message has been triggered.


Test Objectives

This test step examines a Health IT Module's ability to create an ADT^A03 Disposition message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019 conformance requirements.


Evaluation Criteria

Message Validation Report


Notes

Codes are provided in the test data for coding the diagnoses and procedures. The Tool is designed to accept those codes without generating an error. If the HIT developer uses a different but equivalent coded value within the same code system  than the ones provided/accepted, the Tester may ignore errors generated by the Test Tool related to incorrect code/name of coding system when the coded data used in the message are determined via visual inspection to be valid data.

Messages in this test case will be used in a Manual Test Step under functional requirement testing.
