Test Objectives

The Following test objectives are satisfied in this test plan:

This test plan includes examining a Health IT Module's ability to create an ADT^A04-Registration message, an ADT^A08-Update message, an ADT^A03-Discharge message, and an ADT^A01-Admission message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019, conformance requirements.

  • Setting (IP, ED, Urgent Care, Ambulatory)
  • Chief complaint is captured as a coded value in PV2-3 Admit Reason
  • Chief complaint in free-text
  • Diagnoses are submitted test using SNOMED-CT codes:Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the ICD-10 CM and SNOMED CT value set (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). ICD-10 CM codes are provided/required in the Test Data for coding the diagnoses in this Test Case
  • Diagnoses are submitted test using ICD-10-CM codes:Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the ICD-10 CM and SNOMED CT value set (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). ICD-10 CM codes are provided/required in the Test Data for coding the diagnoses in this Test Case
  • Patient Name is Identifiable n the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and pseudonymized data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in a pseudonymized manner is required for this Test Step.
  • Patient name is pseudonymized in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and pseudonymized data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in a pseudonymized manner is required for this Test Step.
  • Multiple races
  • Patient's demographic information is unavailable at registration
  • Updates to Diagnoses
  • Discharge Disposition Home
  • Discharge Disposition Expired
  • Update Diagnosis Post-Discharge
  • Facility sending the syndromic surveillance message is different than the facility where the patient received care
  • Sending required observations where known or applicable 
    • pregnancy, 
    • travel
    • Tobacco use
    • Initial acuity
    • Body Height
    • Body Weight
    • BMI
    • Triage notes
    • Patient Location
    • Chief Complaint
    •  Treating Facility Location
    • Age
      • Pediatric case where age is measured in months.
      • Adult case where age is measured in years.

  • Timeliness
  • Consistency across messages

Evaluation Criteria
null

Notes

 Validation Exceptions:

1. HCPT is an acceptable code (in addition to HCPTNUCC) for OBX-5.3 for the SS003 co-constraint. See the NIST-7 clarification.

2. I10 is an acceptable code (in addition to I10C) for DG1-3.3. See NIST-6 clarification.
