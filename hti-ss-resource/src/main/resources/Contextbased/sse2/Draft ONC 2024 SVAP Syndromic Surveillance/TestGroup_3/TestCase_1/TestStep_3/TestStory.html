Description

At 12:45pm, the body is moved from the inpatient floor to the hospital morgue.  The unit coordinator enters the date/time the body leaves the floor as the discharge date and time.

The hospital's electronic health record module for syndromic surveillance data assembles and transmits an ADT^A03 Discharge message about this encounter to the State Public Health Department (NIST PH SS FAC 2).


NIST Hospital reports inpatient syndromic surveillance data to the State Public Health Department for all new patient discharges at least once per day. At 8:00 PM on June 18, 2025, the hospital's electronic health record module for syndromic surveillance data assembles and transmits an ADT^A03 Discharge message about this encounter to State Public Health Department (NIST PH SS FAC 2). The patient name and birthdate are de-identified in the submission, so no date of birth is included, and the name is represented with name type 'S' to indicate that the name is empty for anonymity.


Comments

The dates and times in this test case illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the test case with EHR technology, only date and time format will be validated within tester submitted test data.


Pre-condition

A01-Admission message is sent before A03-Discharge message.


Post-Condition

No PostCondition


Test Objectives

This test step examines a Health IT Module's ability to create ADT^A03 Discharge message within PHIN Messaging Guide's conformance requirements.

  • Setting is inpatient.
  • Chief complaint is captured as narrative in PV2-3 Admit Reason
  • Patient name is de-identified in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and de-identified data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in a de-identified manner is required for this Test Step.
  • Diagnoses are submitted test using either ICD-10-CM or SNOMED-CT: Per ASTP/ONC, for certification testing  a Health IT Module can use either the ICD-10-CM or SNOMED CT® value sets in the submitted messages for all of the test steps in the Syndromic Surveillance Test Suite. Where the tool does not have test data that supports the Health IT Module’s value set (either ICD-10-CM or SNOMED CT®), the developer of the Health IT Module must provide the codes and testers must perform a visual inspection of the messages for these test steps to ensure that equivalent and valid ICD-10-CM or SNOMED CT® are used to populate the messages.  This policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). ICD-10-CM codes are provided/required in the Test Data for coding the diagnoses in this Test Case.
  • Discharge Disposition Expired
