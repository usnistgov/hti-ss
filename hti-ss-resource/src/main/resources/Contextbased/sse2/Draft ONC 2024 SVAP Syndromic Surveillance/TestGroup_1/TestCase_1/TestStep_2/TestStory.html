Description

At 9:00 AM a nurse sees the patient and performs a vital sign assessment, noting that the child currently has a temperature of 101.2 with a productive cough and right ear inflammation and fluid build-up.

At 9:10 AM, the physician assigns the patient the working diagnoses of influenza with other respiratory manifestations (ICD-10 CM diagnosis code of Influenza due to identified novel influenza A virus with other respiratory manifestations  J09.X2), and Acquired stenosis of external ear canal secondary to inflammation and infection, unspecified ear (ICD-10 CM diagnosis code of H61.329 ) and documents the information in the patient`s electronic medical record. At 9:15  AM the physician orders a rapid influenza test, chest x-ray, and a treatment. The patient`s vital signs gradually return to normal. A nurse weighs the patient, measures his length, and charts the data in the patient’s record. This information is used by the physician when ordering the take-home prescription.

 NIST Hospital's Urgent Care is an outpatient facility operated by NIST Hospital that routinely sends electronic syndromic surveillance data to the the City Health Department (NIST PH SS FAC 1) in accordance with a city regulation. At 9:15 AM, NIST Hospital`s electronic health record module for syndromic surveillance data assembles and transmits an Update ADT message with the working diagnoses and the height and weight information for this patient encounter on behalf of the Urgent Care facility as part of an automated process that transmits new or updated content every 15 minutes.  The patient name and birthdate are de-identified in the submission, so no date of birth is included, and the name is represented with name type 'S' to indicate that the name is empty for anonymity.


. 









Comments

Dates and times are provided in this test scenario to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the test scenario with HIT, only date and time format will be validated within tester submitted test data


Pre-condition

A04-Registration message is sent before the A08-Update message.


Post-Condition

Patient record is updated with procedure results, diagnoses and patient height/weight. An ADT*A08 message has been generated with this updated content.


Test Objectives

This test step examines a Health IT Module's ability to create ADT^A08 Update message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019 conformance requirements.

The following test objectives and variants are supported by this test case:

  • Setting is Urgent Care.
  • Chief complaint is submitted as free-text.
  • Patient name is de-identified in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and de-identified data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in a de-identified manner is required for this Test Step.
  • Diagnoses are submitted test using either ICD-10-CM or SNOMED-CT: Per ASTP/ONC, for certification testing  a Health IT Module can use either the ICD-10-CM or SNOMED CT® value sets in the submitted messages for all of the test steps in the Syndromic Surveillance Test Suite. Where the tool does not have test data that supports the Health IT Module’s value set (either ICD-10-CM or SNOMED CT®), the developer of the Health IT Module must provide the codes and testers must perform a visual inspection of the messages for these test steps to ensure that equivalent and valid ICD-10-CM or SNOMED CT® are used to populate the messages.  This policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019). ICD-10-CM codes are provided/required in the Test Data for coding the diagnoses in this Test Case.
  • Pediatric case where age is measured in months.
  • Updates to Diagnoses
  • Multiple Races
  • Insurance is known
