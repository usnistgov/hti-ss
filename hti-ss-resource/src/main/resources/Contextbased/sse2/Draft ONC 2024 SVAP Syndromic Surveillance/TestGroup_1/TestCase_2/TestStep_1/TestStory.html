Description

62-year-old male walks into NIST Urgent Care on October 20, 2024 at 12:00 PM seeking care for gastrointestinal issues following exposure hurricane-related flood waters. The patient is registered by a clerical assistant who records the patient's name, date of birth, residence information, insurance information (PlanID123 from MyInsuranceCompany), race, ethnicity, smoking status as ex-smoker, and patient's chief complaint, which is “vomiting, diarrhea, nausea, and abdominal pain ". 

NIST Urgent Care is an independently operated outpatient facility that routinely sends electronic syndromic surveillance data to the State Department of Public Health (NIST PH SS FAC 2) in accordance with a state regulation. The state regulations permit sending identifiable patient information for the purposes of syndromic surveillance. At 12:15 PM, the facility`s electronic health record module for syndromic surveillance data assembles and transmits a Registration ADT message about this patient encounter. The "full, legal patient name" and birth date is included due to state required reporting of all patients, which is not always the case across all jurisdictions.  






Comments

Dates and times are provided in this test scenario to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the test scenario with HIT, only date and time format will be validated within tester submitted test data. 


Pre-condition

No Pre-Condition


Post-Condition

An ADT^A04 Registration message has been triggered.


Test Objectives

This test step examines an HIT Module's ability to create an ADT^A04 Registration message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019, conformance requirements.

The following test objectives and variants are supported by this test case:

  • Setting is Urgent Care.
  • Chief complaint is captured as a narrative in PV2-3 Admit Reason.
  • Patient name is identifiable in the submissions: Per ASTP/ONC, for certification testing HIT developers must demonstrate that their system supports the Patient Name and pseudonymized data in PID-5 (this policy is conformant to the named standard HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019).The patient name presented in an identified manner is required for this Test Step.
  • Adult case where age is measured in years.
  • Insurance is unknown

Evaluation Criteria

Message Validation Report


Notes

Messages in this test case will be used in a Manual Test Step under functional requirement testing.
