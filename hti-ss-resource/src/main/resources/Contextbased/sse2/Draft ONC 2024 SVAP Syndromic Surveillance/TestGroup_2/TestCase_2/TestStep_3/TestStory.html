Description

Upon review of the imaging results, the physician determines the newborn should be admitted to the hospital NICU and put on ECMO to stabilize the baby before performing surgery to repair the CDH (0BQT0ZZ Repair Diaphragm, Open Approach).

 The patient is quickly discharged from the ED and transported within the facility to the NICU . At 8:25 AM, ED staff complete the patient record and administratively discharge the patient from the ED. The working diagnosis is updated to final diagnosis. The patient's final diagnosis is (Pericardial and diaphragmatic defect syndrome (disorder) (SNOMED CT code 724068001)),  NIST Hospital reports syndromic surveillance data to the City Health Department (NIST PH SS FAC 1). At 8:30 AM on February 1, 2025, the hospital's electronic health record module for syndromic surveillance data assembles and transmits a Discharge message about this encounter. The "full, legal patient name" and birth date is included due to state required reporting of all patients, which is not always the case across all jurisdictions.  


Comments

Dates and times are provided in this test case to illustrate the sequence of clinical and messaging events. Since the exact dates and times are not reproducible when modeling the Test Case with an HIT Module, only date and time format will be validated within tester submitted test data.


Pre-condition

A08-Update message is sent before A03-Discharge message.


Post-Condition

No PostCondition


Test Objectives

This test step examines an HIT Module's ability to create an ADT^A03 Discharge message within HL7 Version 2.5.1 Implementation Guide: Syndromic Surveillance, Release 1 - US Realm Standard for Trial Use July 2019 conformance requirements.


Evaluation Criteria

Message Validation Report


Notes

Codes are provided in the test data for coding the diagnoses and procedures. The Tool is designed to accept those codes without generating an error. If the HIT developer uses a different but equivalent coded value within the same code system  than the ones provided/accepted, the Tester may ignore errors generated by the Test Tool related to incorrect code/name of coding system when the coded data used in the message are determined via visual inspection to be valid data.

Messages in this test case will be used in a Manual Test Step under functional requirement testing.
